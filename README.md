# そのフロート、アイス乗ってません！

エンジニアと非エンジニアで意味の食い違う語を、**両方の視点から並べて引ける辞書**です。

> 「ここ float で持ってます」と言われた非エンジニアが思い浮かべているのは、
> だいたいメロンソーダの上でゆっくり溶けていくアイスクリームである。

## エンジニア度スライダー

このサイトの核となる仕掛けです。エンジニア側の説明文にはスライダーが付いていて、
引き下げると専門的な語句が順にフェードアウトし、文字が詰まって平易な文へ変化します。

```
エンジニア度 10   IEEE 754 で定められた、符号部・指数部・仮数部により浮動小数点数を 32bit で表現するデータ型。
エンジニア度  8   符号部・指数部・仮数部により浮動小数点数を 32bit で表現するデータ型。
エンジニア度  5   浮動小数点数を 32bit で表現するデータ型。
エンジニア度  3   浮動小数点数を表現する型。
エンジニア度  0   小数。
```

最後に残るのは「小数。」です。エンジニアが日頃 float と呼んでいるものの正体であり、
「こいつら、小数のことをフロートとか言ってるのか」と気づいてもらうのが狙いです。

一般側の説明はスライダーでは変化しません。エンジニア側だけが崩れていき、
最終的に一般側の隣に並ぶ構図をつくります。

## 語を追加する

`src/content/words/<語>.md` を作ります。

```yaml
---
term: "float"
reading: "ふろーと"
aliases: ["フロート", "浮動小数点数"]

engineer:
  category: "データ型"
  levels:
    - "IEEE 754 で定められた、符号部・指数部・仮数部により浮動小数点数を 32bit で表現するデータ型。"
    - "符号部・指数部・仮数部により浮動小数点数を 32bit で表現するデータ型。"
    - "浮動小数点数を 32bit で表現するデータ型。"
    - "浮動小数点数を表現する型。"
    - "小数。"

general:
  category: "飲料"
  meaning: "アイスクリームを浮かべた炭酸飲料。クリームソーダなど。"

tags: ["初心者殺し", "飲食店で言うな"]
publishedAt: 2026-07-23
---

本文（任意）。もう少し長い解説やエピソードを書けます。
```

`levels` は先頭ほどエンジニア寄りです。段階の数は語ごとに変えてかまいません。
ビルド時に 0〜10 の共通スケールへ自動で配分されます。

### 執筆ルール：levels は「削るだけ」

**各段階は、直前の段階から文字を削除するだけで到達できなければなりません**（部分列であること）。
語順の入れ替えや助詞の置換はできません。

```yaml
# NG: 「を使って」を「で」に置換している
- "32bit を使って表現する。"
- "32bit で表現する。"

# OK: 削除だけで到達できる
- "32bit を使って実数を表現するデータ型。"
- "実数を表現する型。"
```

一見きびしい制約ですが、これによってページに出力される説明文は
**最上位段階の全文ひとつだけ**で済みます。段階ごとに別々の文を埋め込む必要がなく、
検索エンジンには完全な全文がインデックスされ、消える箇所だけが `<span>` に
切り分けられます。

制約を破るとビルドが止まり、どのファイルのどこが問題か表示されます。

```
[InvalidContentEntryDataError] words → float data does not match collection schema.
  engineer.levels: levels[1] が levels[0] の部分列になっていません。
  文字の削除だけで到達できる必要があります（助詞の置換は不可）。
```

原稿を書くときは、**削っても助詞が繋がるように最初から設計する**のがコツです。

## 仕組み

説明文の変形に JavaScript はほとんど使っていません。

1. ビルド時に隣接段階の差分を取り、消える文字を `<span data-show="…">` で包む
2. スライダーは `<html data-eng="N">` を書き換えるだけ
3. 表示の切り替えは CSS の属性セレクタが担当する

```css
html[data-eng="7"] [data-show]:not([data-show~="7"]) {
  display: none;
}
```

`<html data-eng>` が未設定のとき（JavaScript 無効時）はどのルールにもマッチしないため、
もっともエンジニア寄りの全文がそのまま読める状態になります。

## ディレクトリ構成

```
src/
  content/words/          語ごとの Markdown
  content.config.ts       スキーマと部分列制約の検証
  lib/
    subsequence.ts        部分列判定
    levelize.ts           差分 → 断片の切り分け、共通スケールへの配分
  components/
    EngineerSlider.tsx    エンジニア度スライダー
    LevelText.astro       断片の描画
  layouts/Layout.astro
  pages/
    index.astro           語の一覧
    words/[...id].astro   語の詳細
  styles/global.css       表示切替の CSS
docs/
  tech-selection.md       技術選定の全体像と決定理由
  progress.md             現在地・残タスク
```

## 技術スタック

Astro 7 / React 19（islands のみ）/ TypeScript / Tailwind CSS v4 / nanostores

完全な静的サイトです。サーバーサイドの実行環境を必要としません。

## デプロイ

さくら VPS（Ubuntu）+ nginx で静的配信します。ローカルでビルドして `dist/` を
rsync する方式のため、サーバー側に Node.js は不要です。
（デプロイ先は未設定。`docs/tech-selection.md` を参照）
