**`float` 1 語が動く状態。** 意味は 2 枚の重なるカードで表示し、エンジニアカードの
3 段階チップ（さらっと / しっかり / がっつり）で説明文を切り替える。

### データ構造：タグと用例

**タグ＝分野ファセット。「何の分野の語か」だけを担う**（絞り込みと回遊の軸）。
性質・ネタ・すれ違いの型はタグに載せない ── それらは本文で語る。この役割分担が肝。

- 1 語は 2 つの世界にまたがるので、タグも 2 世界に分かれる。
  `src/lib/tags.ts` の `TAGS` レジストリで、各タグに `world`（`tech` = エンジニア世界 /
  `daily` = 日常世界）と `description` を持たせる。float なら tech の `データ` と daily の `飲料`
- `world` は表示色にも使う。**tech = エンジニア緑（`bg-engineer`）/ daily = ふつうピンク（`bg-general`）**。
  `#データ`（緑）と `#飲料`（ピンク）が並ぶだけで「両方の世界に属する」というサイトの前提が一目で伝わる
- 統制語彙。`z.enum(TAG_NAMES)` で検証し、**未定義タグはビルドを止める**（部分列制約と同じ思想）。
  検証済み：不正タグを置くと許可語の一覧つきでエラーが出る
- 語ページ・一覧の両方でタグを表示（同じレジストリを引く）。`title` に description
- **`category` は廃止**した（`engineer.category` / `general.category`）。分野はタグへ一本化。
  パネル見出しは「エンジニア」「ふつう」だけになった

**用例**：`engineer.examples` / `general.examples`（`string[]`, 既定 `[]`, 目安 2〜3 個）。
前面カードの本文下に「用例」として表示（`MeaningCards.tsx` 内でレンダリング）。
段階切替とは無関係な静的テキスト。

> タグ設計の経緯：当初は分野＋性質＋ネタを 1 配列に混ぜていたが、軸が混在して
> 分かりにくかった。「タグは分野だけ／それ以外は本文」に整理して役割を一意にした。

### 意味カード（このプロジェクトの核）

`src/components/MeaningCards.tsx` が唯一の状態を持つ島。

- **重なる 2 枚**：**グリッドの同じセルに重ねる**（`col-start-1 row-start-1`）。
  前面は `rotate-0` ＋ 影 10px（ホバーで左上へ 1・影 14px）、背面は右下へ `translate-4`
  （sm では 6）＋ `rotate-3` ＋ 影 4px。位置と影は参考の値そのまま。
  参考は `absolute inset-0` だが、それだとカードが固定高になって本文があふれるので
  グリッド重ねにした。**セルの高さは背の高い方に合う**ので、長い段階を選んでも
  カードが伸びて収まる（スクロールバーが出ない）。幅は `320px` / sm `560px`、
  高さは `min-h-[280px]` / sm `min-h-[300px]` の下限指定
- **付箋（旧タブ）**：角ばった形＋`rotate-[±2deg]`＋影 3px で、手で貼った付箋に見せる。
  `-top-16 sm:-top-20 h-18 sm:h-22` の `z-[-1]`。ENGINEER が右、GENERAL が左。
  **高さと `-top-*` は「背面に回ってもラベルが前面カードの上端より上に残る」条件で
  決めている**（背面は右下へ 16/24px ずれるため、浅いとラベルが前面カードに食われる）。
  コンテナの `mt-24` が付箋の逃げ
- **前後入替**：**カードのどこをクリックしても入れ替わる**（参考と同じ）。
  `duration-500` ＋ `cubic-bezier(0.34,1.56,0.64,1)` で行き過ぎて戻る。
  段階チップと下段バッジは `stopPropagation` して入替を起こさない。
  キーボードは下段バッジ（`button`）で操作でき、背面側のバッジは `tabIndex={-1}` で外す
- **下段バッジは 2 枚で共通**（`Badge` コンポーネント）。形も並びも同じで、文言だけが
  `TAP FOR ENGINEER` / `TAP FOR GENERAL` と行き先で入れ替わる。日本語の読み上げ用に
  `aria-label` は「〜の意味を前面に出す」を別に持たせている
- **2 枚は同じ骨格**：見出しチップ（アイコン＋語）／チップ列／本文／下段
  （透かし＋バッジ）。参考の ENGINEER カード側の構造に GENERAL を合わせた。
  当初は参考の名刺カード（大見出し＋アバター＋アイコン 2 つ）をそのまま
  GENERAL に当てていたが、2 枚のデザインが違って見えるのでやめた。
  違いは色・付箋・アイコン（`Terminal` / `Smile`）・チップ列・本文だけで、
  構造は `card()` 一か所。透かしとバッジの行き先表示は `CARDS.label` の使い回し
- **3 段階切替**：参考の技術スタックのチップ列が段階ボタンにあたる。
  白チップ＝非選択、黒チップ＝選択中。初期は しっかり（index 1）。
  RetroUI の `Button` は使っていない（参考のチップの見た目を優先）。
  GENERAL 側は切替がないので、同じ位置に読みの白チップを 1 つ置いて骨格を揃えている
- **本文はスクロールさせない**：カードの高さが伸びるので、がっつりのような長い段階も
  そのまま収まる。以前は固定高＋`overflow-y-auto`＋マスクフェードだったが廃止した
- **用例はカードの外**（島の下）に置き、前面の側に追従して入れ替わる。
  カードの中には入らない（参考のレイアウトに用例の枠がないため）
- **状態はカードのローカル**（`useState`）。`client:load` で SSR＋ハイドレーション
- 一覧ページは**同じコンポーネントに `href` を渡した静的プレビュー**（島にしない）。
  カード全体が語ページへのリンクになり、ボタン類は `span` で出る。
  前面は語ページと同じ ENGINEER 側

### 設計の変遷（旧「核」の廃止）

当初は **部分列制約＋CSS だけで説明文を段階変形**する方式だった（`engineer.levels` の後段が
前段の部分列で、消える文字を `<span data-show>` に切り、`html[data-eng=N]` で表示制御）。
これを 2026-07-24 に全面的にやめた。

- **理由**：部分列を保ったまま自然で多段階な説明を書くのが執筆上つらい。
  自由に 3 段階を書けるほうが語を増やしやすい
- **廃止したもの**：`subsequence.ts` / `levelize.ts` / `LevelText.astro` /
  `EngineerSlider.tsx` / `stores/engineerLevel.ts` / `ui/slider.tsx` / global.css の
  `data-eng`/`data-show` ルール / Layout のちらつき防止スクリプト
- **`engineer.levels` は「3 個ちょうど」の独立した全文**に。`content.config.ts` が
  `length(3)` で検証し、3 個でなければビルドが止まる（検証済み）

## 保留中の判断

### カードの高さ（解決済み・経緯だけ残す）

参考のカードは名刺サイズ（340×210 / sm 400×240）の固定高で、**元は英語の短い行を
3〜4 行置くための寸法**。日本語の説明を入れると本文が 2〜3 行しか入らず、
がっつり（130 字前後）はカード内スクロールになっていた。

**2 枚をグリッド重ねにして高さを可変**（下限のみ指定）にし、寸法も一回り
大きくして解決した。段階を切り替えるとカードの高さが変わるが、入替と同じ
トランジションに乗るので不自然さはない。

ヘッドレス Chrome で 390px 幅・700px 幅、エンジニア前面／ふつう前面、
さらっと〜がっつりを確認済み（横スクロールなし、切れなし、用例の追従あり）。

### 一覧ページの前面

一覧は語ページと同じ ENGINEER 前面にしてある（語名は両カードの見出しチップに
出るので、どちらを前にしても語は読める）。背面側の説明は一覧では読めないため、
ふつうの意味を一覧で見せたいなら前面を入れ替えるか、両方を並べる作りに変える。

### ロゴ右端が元画像の時点で切れている

バッジ右側のネイビー枠が、元画像のキャンバス右端で縦 200px ぶん途切れている。
透過処理の副作用ではなく、渡された画像がすでにそうなっていた。
「枠がフチまで抜けているデザイン」としても読めるため現状維持にしてあるが、
気になるなら余白を付けて生成し直すのが確実。

### 落とし穴：dev サーバーの CSS が古いままになる

**新しい Tailwind クラスを足した直後にレイアウトが崩れたら、まず dev サーバーを
再起動する。**（2026-07-24 に実際に踏んだ）

- 症状：ホーム → 語ページとページ内遷移したときだけ崩れ、リロードすると直る。
  幅・高さ・位置の指定だけが効かず、色や枠線は効いている、という出方をする
- 原因：Tailwind v4 は使われているクラスを見つけて CSS をオンデマンド生成する。
  HMR は**今表示中のページのスタイルにパッチを当てる**方式なので、`<ClientRouter />`
  のページ内遷移でヘッドが差し替わると古い方が勝つ。実測でも、遷移後に読まれている
  CSS にセッション中に追加した `h-22` などが 1 つも入っていなかった
- 特に `h-18` / `h-22` のようなスケール外の値や `w-[320px]` のような任意値で出やすい
- **本番ビルドでは起きない**。両ページとも同じハッシュ付き CSS を 1 本参照するため。
  `npm run preview` で「ホーム→語ページ」「語→ホーム→語」「戻るボタン」を実際に
  遷移させて確認済み
